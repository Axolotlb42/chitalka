# Читалка (Kivy + PyMuPDF) — пример приложения для Android

Это простая «читалка» учебников на Python с использованием Kivy. Приложение:
- Ищет PDF в папке Books (несколько стандартных мест, включая /sdcard/Books).
- Показывает список файлов.
- Открывает PDF внутри приложения, рендерит страницы в PNG с помощью PyMuPDF (fitz) и кеширует их.
- Управление: вперед/назад, ввод номера страницы.

Важно:
- Для рендеринга PDF нужно PyMuPDF (пакет `PyMuPDF`, импортируется как `fitz`).
- На Android при сборке через Buildozer/ p4a убедитесь, что в requirements добавлены `pymupdf` и `kivy`.
- На Android нужно разрешение на чтение внешнего хранилища (READ_EXTERNAL_STORAGE). В buildozer.spec добавлено соответствующее разрешение, но для Android 6+ нужно также запрашивать runtime permission (в коде есть попытка request_permissions).

Куда поместить учебники:
- Поместите PDF-файлы в одну из следующих папок:
  - /sdcard/Books
  - /storage/emulated/0/Books
  - или рядом с приложением (папка Books в каталоге с main.py, для теста на ПК)
  - ~/Books

Сборка:
1. Установите buildozer (Linux) и настройте buildozer.spec (пример ниже).
2. В buildozer.spec добавьте в requirements: kivy, pymupdf, pillow (если нужно).
3. Соберите APK: `buildozer android debug` (или подходящая команда для вашей системы).

Ограничения и советы:
- PyMuPDF — не самый лёгкий пакет для Android, сборка может быть долгой. Если у вас сложности со сборкой PyMuPDF, рассмотрите альтернативы: серверный предрендер страниц или использование Android-native PDFRenderer через pyjnius.
- Приложение кеширует PNG-изображения страниц в user_data_dir (папка приложения) чтобы не рендерить заново.
- Это пример — вы можете улучшить интерфейс (жесты масштабирования, скроллинг страниц, поиск и т.д.).

Файлы:
- main.py — основной код приложения (Kivy + PyMuPDF)
- buildozer.spec — примерный spec для сборки (ниже)
